JavaScript FUNCTION Syntax

A JS function is defined with the function keyword, followed by a name, followed by parantheses ().
Functions names can contain letters, digits, underscores and dollar signs (same rules as VARIABLES).
The parameters may include parameter names separated by commas: (p1, p2).

Functions parameters are LISTED inside the parantheses () in the funciton DEFINITION.
Function arguments are the values received by the FUNCTION when it is invoked.
Inside the function, the arguments (the parameters) behave as LOCAL VARIABLES.

Function Invocation

The code inside the function will execute when "something" - invokes (calls) the function.

  • When an event occurs (when a user CLICKS a BUTTON)
  • When it is INVOKED (called) from JavaScript code
  • Automatically (self invoked)

Function RETURN

When JS reaches a return statement, the FUNCTION will stop executing.
If the function was invoked from a stetement, JS will "RETURN" to execute the code after the invoking statement.
Functions often compute a return value.

	var res = str.slice(-12);

	parseInt()/parseFloat() - these are METHOD that exist @ the GLOBAL lvl - within the JS.
	So - we can call parseInt() to take any string "123" (like this) - or a number that HAS a DECIMAL (like this) 123.456.
	Take a "float" or a "string" and EXTRACT the INTEGER part of it.

	let myStrInt = "456";
	console.log(parseInt(myStrInt)); //456

	456 is NOW a NUMBER. (When you receive a string number from a ..idk user has typed someplace else "that value" ... you need to deal with it as if it were a NUMBER; but the value you get it is a STRING.) We need to convert it from the STRING into a NUMBER - what the "PARSE " methods are doing.
	parseInt() - takes the STRING and turns it into a NUMBER .

	ANOTHER Example:
	let address = "456 Main Ave."
	console.log(parseInt(address)); //456;
	It looks at this string - starts at the beginning - yeap, that's a number, that's a number, that's a number - OH ok...SPACE - that's not a number. I will STOP here.
	
	IF we do:
	console.log(parseFloat(address)); // 456.
	we get the same VALUE. iT DOESN'T force TO BE a DECIMAL. parseFloat() will give you back an INTEGER. (YOU NEED TO HAVE a DECIMAL for "me" to work!)
	let myStrFloat = "345.567.789.345";
	console.log(parseFloat(myStrFloat)); //345.567
	we exctract 345.567 and that's where it stops - because you cannot have a MULTIPLE DECIMALS within a number.
